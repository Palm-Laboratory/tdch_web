# 담임목사 소개 페이지 작업 계획서

## 1. 작업 개요

- 대상 페이지: `/about/pastor`
- 메뉴 위치: `교회소개 > 담임목사 소개`
- 현재 상태: 플레이스홀더 화면만 존재
- 목표: 현재 프로젝트 구조와 반응형 원칙을 유지하면서, 웹 기준으로 설계된 5개 섹션 페이지를 모바일 베이스 코드로 구현

## 2. 현재 프로젝트 기준 정리

### 라우팅 및 레이아웃

- `src/app/about/pastor/page.tsx`
  - 현재는 제목과 임시 박스만 있는 상태
- `src/app/about/layout.tsx`
  - `교회소개` 공통 `PageHeader` + `Breadcrumb`를 사용
  - `/about/greeting`만 예외적으로 상단 헤더를 숨김
- `src/lib/site-data.ts`
  - `교회소개` 하위 메뉴에 `/about/pastor`가 이미 연결되어 있음

### README 기준 제약

- 기본 작성 기준은 모바일 우선
- 레이아웃 변화는 가능하면 `md`, `lg`에서만 처리
- `xl`은 와이드 화면 보정만 허용
- 새 코드에서는 `sm`, `2xl` 사용 지양
- 색상은 `globals.css` CSS 변수와 Tailwind 팔레트를 우선 재사용

## 3. 현재 상태 진단

- 페이지 엔트리는 이미 존재하므로 신규 라우트 생성은 불필요
- `about` 공통 레이아웃 아래에 페이지 콘텐츠만 교체하면 됨
- 현재 Figma 구조 분석은 MCP 한도 문제로 불가했으므로, 사용자 설명 기반으로 레이아웃을 해석해야 함
- 따라서 이번 작업은 "웹 시안 의도 유지 + 모바일 베이스 재구성"이 핵심

## 4. 구현 방향

### 기본 방향

- 웹 기준 5개 섹션을 하나의 긴 스크롤 페이지로 구성
- 모바일에서 먼저 자연스럽게 읽히는 단일 컬럼 구조로 설계
- 데스크톱에서는 `lg` 이상에서만 가로 배치, 여백 확장, 비주얼 강조를 적용
- 기존 `about` 영역의 헤더/브레드크럼/톤과 충돌하지 않도록 페이지 내부 스타일을 조정

### 예상 구성 방식

- 상단 인트로 블록
  - 페이지 제목, 영문 보조 텍스트, 짧은 소개문 또는 대표 메시지
- 본문 5개 섹션
  - 각 섹션은 독립적인 `section` 블록으로 분리
  - 배경 톤, 이미지 유무, 텍스트 정렬 방식이 다를 가능성에 대비해 재사용 가능한 래퍼 구조로 설계
- 필요 시 공통 유틸 추출
  - 섹션 타이틀 블록
  - 강조 문장 블록
  - 프로필/이력 리스트 블록
  - 이미지 + 텍스트 2단 조합 블록

## 5. 작업 단계

### 1단계. 요구사항 구체화

- 사용자가 설명하는 5개 섹션의 목적, 순서, 핵심 문구를 정리
- 각 섹션별로 아래 항목 확인
  - 섹션 제목
  - 본문 유형
  - 이미지 필요 여부
  - 강조 요소
  - 웹에서의 배치 의도

### 2단계. 정보 구조 설계

- 5개 섹션을 페이지 흐름에 맞게 정렬
- 모바일 기준 읽기 순서를 먼저 확정
- 데스크톱에서만 추가할 레이아웃 전환 지점을 정의

### 3단계. 컴포넌트 분리 여부 판단

- 섹션 구조가 반복되면 `src/app/about/pastor/components/` 또는 인접 경로로 분리
- 반복이 약하면 `page.tsx` 내부에서 우선 구현
- 최종 판단 기준
  - 재사용성
  - 파일 길이
  - 섹션 간 스타일 차이

### 4단계. 스타일 시스템 매핑

- 기존 팔레트와 타이포를 우선 사용
- Figma 색상값이 추후 필요하면 프로젝트 토큰에 맞춰 근사 매핑
- 여백 체계는 `section-shell` 및 현재 `about` 페이지 톤과 맞춤

### 5단계. 반응형 구현

- `base`: 단일 컬럼, 읽기 흐름 우선
- `md`: 간격 확대, 일부 2단 전환 가능
- `lg`: 웹 기준 핵심 배치 반영
- `xl`: 필요 시 폭/타이포만 미세 보정

### 6단계. 검수

- `/about` 공통 헤더와 시각적 충돌 여부 확인
- 모바일/태블릿/데스크톱에서 섹션 간 리듬 점검
- 임시 문구/이미지 경로 누락 여부 점검

## 6-1. Section 1 상세 반영

### 섹션 역할

- 페이지 첫 인트로 섹션
- 담임목사 이름, 영문명, 직함, 첫 인사 문장을 함께 전달하는 대표 비주얼 구간
- 웹 시안에서는 좌측 텍스트, 우측 인물 이미지의 2단 구성

### 배경 처리

- 기본 배경은 진한 남색 계열의 선형 그라디언트
- Figma 기준:
  - `Linear 0%`: `#111C36` 100%
  - `Linear 70%`: `#111C36` 0%
- 실제 구현에서는 어두운 남색 바탕 위에 우측으로 갈수록 은은하게 밝아지는 톤으로 해석

### 우측 배경 오브젝트

- 우측 영역에 Ellipse 3개 배치
- 각 Ellipse 공통 효과
  - `Layer blur: 120`
  - `Radial Fill`
    - `0%`: `#6D7FA8` 24%
    - `100%`: `#6D7FA8` 0%
- 구현 시 absolute 장식 요소로 배치
- 모바일에서는 가독성을 해치지 않도록 일부 축소/재배치 필요

### 이미지

- 사용 이미지: `public/images/about/senior_pastor.png`
- 웹 시안 기준 우측 배치
- 모바일에서는 텍스트 아래로 내려오는 단일 컬럼 전환 우선

### 좌측 텍스트

- 시안 문구를 그대로 반영
- 현재 확인 가능한 구성
  - 상단 작은 라벨: `담임목사 소개`
  - 메인 이름: `이진욱 목사`
  - 영문명: `Timothy Lee`
  - 직함 라벨 박스: `THE DISCIPLES CHURCH · SENIOR PASTOR`
  - 본문
    - `솔직히, 목사 소개 페이지는 좀 어색하죠.`
    - `자랑처럼 보일 것 같고, 너무 거룩하게 써놓으면`
    - `오히려 멀게 느껴지고.`
    - `그래서 그냥 있는 그대로 써보려고 합니다.`

### 구현 메모

- 한글 이름은 프로젝트의 로컬 서체 `YeongwolTTF`를 우선 검토
- 영문명과 보조 라벨은 serif 계열을 사용해 시안 톤 유지
- 금색 라인/보더/텍스트는 기존 팔레트와 충돌하지 않는 범위에서 시안값에 가깝게 적용
- PC 기준 Section 1 실사이즈는 `1120 x 474`로 맞춤
- 1차 구현 범위는 Section 1 완성까지
- 이후 Section 2~5 설명을 받으며 페이지 전체 조합으로 확장

- 새 페이지를 만드는 작업이 아니라 기존 `/about/pastor` 내용을 교체하는 작업으로 진행
- 모바일 베이스 원칙은 유지하되, 사용자가 설명하는 웹 기준 레이아웃 의도를 `lg` 이상에서 반영
- 섹션 수는 5개로 고정하고, 상세 구조는 다음 설명을 받아 확정
